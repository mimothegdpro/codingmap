Rush Battles - Roblox Game

A weapon battle game where players roll for weapons and fight each other. Built the entire map from scratch. Learned a painful lesson about backups when a GUI malfunction destroyed everything.

Luau Roblox Studio Game Dev

Rush Battles was my passion project. Players would spawn in, roll for a random weapon, and battle each other in the map I designed. I poured hours into building the terrain, scripting the weapon system, and creating the GUI. Then disaster struck - a bug in the GUI system caused everything to get deleted. 124 lines of code... gone. It was heartbreaking, but it taught me the most important lesson in programming: ALWAYS. BACKUP. YOUR. CODE.

Why I Code

Coding is more than just typing commands into a computer - it's about creating something from nothing. Every line of code I write is a piece of magic that brings my ideas to life. When I first made that simple dice roller, I realized I could build ANYTHING I imagine. That feeling never gets old.

Yes, I've had setbacks - like losing 124 lines of Rush Battles code. But that taught me resilience. Real developers don't give up when things break; they learn, adapt, and come back stronger. Every bug is a puzzle, every error is a lesson, and every project is a step closer to my dream.

I code because I want to build things that matter. I want to create websites that people love, games that bring joy, and applications that solve real problems. When I grow up and work for big tech companies, I'll remember that kid who started with a 3-line dice roller at 12 years old. That's my story, and I'm just getting started.
